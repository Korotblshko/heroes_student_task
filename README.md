# Проект Heroes Battle - Алгоритмы и структуры данных


## Реализованные алгоритмы

### 1. GeneratePreset - генерация армии противника
Использован жадный алгоритм: юниты сортируются по соотношению атака/стоимость, затем здоровье/стоимость.
Добавляются максимально возможное количество юнитов каждого типа (до 11), пока хватает очков.

Алгоритмическая сложность: O(t log t), где t = 4 (число типов юнитов) - лучше требуемой.

### 2. SimulateBattle - симуляция боя
Каждый раунд:
- Собираются все живые юниты
- Сортируются по убыванию атаки
- По очереди делают ход (attack())
- После атаки выводится лог
- Погибшие юниты исключаются

Сложность одного раунда: O(n log n), где n - число юнитов.

### 3. SuitableForAttackUnitsFinder - поиск доступных целей
Для каждого ряда выбирается открытый юнит:
- При атаке слева - самый левый (мин Y)
- При атаке справа - самый правый (макс Y)

Сложность: O(n) - линейная.

### 4. UnitTargetPathFinder - поиск пути
Использован BFS (поиск в ширину)
Препятствия - живые юниты (кроме цели).

Сложность: O(W × H) = O(567) - оптимальная для задачи.
